# 手动安装指南 - 容器化编程环境配置

> **适用场景**：当自动化脚本 `Install.bat` 失效或中断时，可按本指南手动完成环境配置。

---

## 📋 前置准备

### 系统要求
- **操作系统**：Windows 10 (版本 1903+) 或 Windows 11
- **硬件要求**：
  - 剩余磁盘空间 > 20GB
  - CPU 已在 BIOS 中开启**虚拟化技术** (Intel VT-x / AMD-V)
- **管理员权限**：所有操作均需以管理员身份运行 PowerShell 或命令提示符

### 检查虚拟化是否开启
1. 按 `Win + R`，输入 `taskmgr` 打开任务管理器
2. 切换到"性能"选项卡 → 点击"CPU"
3. 确认右下角显示"虚拟化: 已启用"
4. 如未启用，需重启电脑进入 BIOS 设置开启 VT-x/AMD-V

## 第一步：配置 WSL2 环境

### 1.1 启用 Windows 功能

以**管理员身份**打开 PowerShell，执行以下命令：

```powershell
# 启用 WSL 功能
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

**执行后需要重启电脑**，重启后继续下一步。

### 1.2 安装 WSL2 内核更新包

#### 方法 A：使用本地资源（推荐）

如果 `Resources` 文件夹中存在 `wsl_update_x64.msi` 或 `wsl_update_arm64.msi`：

```powershell
# 进入项目目录
cd D:\WinDevEnvSetup

# 根据架构选择对应文件安装（x64 系统使用 x64，ARM 系统使用 arm64）
msiexec.exe /i "Resources\wsl_update_x64.msi" /quiet /norestart
```

#### 方法 B：通过网络更新

```powershell
wsl --update
```

### 1.3 设置 WSL2 为默认版本

```powershell
wsl --set-default-version 2
```

### 1.4 （可选）修改 WSL 默认存储路径

默认情况下 WSL 安装在 C 盘，建议迁移到 D 盘以节省空间：

```powershell
# 创建目标目录
New-Item -ItemType Directory -Path "D:\WSL" -Force

# 设置环境变量（需要管理员权限）
[Environment]::SetEnvironmentVariable('WSL_INSTALL_PATH', 'D:\WSL', 'Machine')
```

**注意**：此步骤仅影响后续安装的 WSL 发行版，已安装的需手动迁移。

## 第二步：安装 Scoop 包管理器

### 2.1 设置 Scoop 安装路径

以**管理员身份**打开 PowerShell：

```powershell
# 设置用户级安装路径
$env:SCOOP = 'D:\Scoop'
[Environment]::SetEnvironmentVariable('SCOOP', 'D:\Scoop', 'User')

# 设置全局安装路径
$env:SCOOP_GLOBAL = 'D:\ScoopGlobal'
[Environment]::SetEnvironmentVariable('SCOOP_GLOBAL', 'D:\ScoopGlobal', 'Machine')
```

### 2.2 安装 Scoop（使用国内镜像）

```powershell
# 允许执行脚本
Set-ExecutionPolicy RemoteSigned -Scope Process -Force

# 通过国内镜像安装 Scoop
Invoke-Expression "& {$(Invoke-WebRequest -useb scoop.201704.xyz)} -RunAsAdmin"
```

### 2.3 配置文件夹权限

```powershell
# 获取当前用户名
$CurrentUser = [System.Security.Principal.WindowsIdentity]::GetCurrent().Name

# 授予完全控制权限
icacls "D:\Scoop" /grant "${CurrentUser}:(OI)(CI)F" /T /Q /C
```

### 2.4 安装基础工具

```powershell
# 安装 7zip 和 git（Scoop 依赖）
scoop install 7zip git

# 配置 Git 安全目录
git config --global --add safe.directory "D:\Scoop/*"
```

### 2.5 添加软件仓库（Bucket）

```powershell
# 移除默认 main bucket（如果存在）
scoop bucket rm main

# 重新添加 main 和 extras bucket
scoop bucket add main
scoop bucket add extras
```

### 2.6 安装核心应用

```powershell
# 安装 Podman 和 VSCode
scoop install podman vscode

# 清理缓存
scoop cache rm *
```

### 2.7 刷新环境变量

```powershell
# 刷新当前会话的环境变量
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
```

**建议**：重启电脑以确保环境变量在所有程序中生效。

## 第三步：配置 Podman 容器引擎

### 3.1 创建数据存储符号链接

将 Podman 数据重定向到 D 盘以节省 C 盘空间：

```powershell
# 创建目标目录
New-Item -ItemType Directory -Path "D:\WSL\podman-data" -Force

# 创建符号链接（需要管理员权限）
cmd /c mklink /j "$env:LOCALAPPDATA\containers" "D:\WSL\podman-data"
```

### 3.2 初始化 Podman 虚拟机

#### 方法 A：使用本地镜像（推荐，离线可用）

如果 `Resources` 文件夹中存在 `podman-machine-*.tar.zst` 文件：

```powershell
# 进入项目目录
cd D:\WinDevEnvSetup

# 查找镜像文件
$rootfs = Get-ChildItem -Path "Resources" -Filter "podman-machine-*.tar*" | Select-Object -First 1

# 使用本地镜像初始化
podman machine init --image-path "$($rootfs.FullName)" --rootful
```

#### 方法 B：通过网络下载（需要联网）

```powershell
# 直接初始化，Podman 会自动下载官方镜像
podman machine init --rootful
```

**注意**：`--rootful` 参数表示以 root 权限运行容器，这对于某些开发场景是必需的。

### 3.3 启动 Podman 虚拟机

```powershell
podman machine start
```

### 3.4 验证 Podman 是否正常工作

```powershell
# 查看虚拟机状态
podman machine list

# 测试运行容器
podman run hello-world
```

## 第四步：导入或构建开发容器镜像

### 4.1 检查虚拟机状态

```powershell
# 确保虚拟机正在运行
$status = podman machine inspect --format "{{.State}}"
if ($status -ne "running") {
    podman machine start
}
```

### 4.2 导入或构建镜像

#### 方法 A：从本地压缩包导入（最快）

如果 `Resources` 文件夹中存在 `arch-cpp-dev.tar.zst`：

```powershell
# 进入项目目录
cd D:\WinDevEnvSetup

# 导入镜像
podman load -i "Resources\arch-cpp-dev.tar.zst"
```

#### 方法 B：从 Dockerfile 构建（需要联网）

如果 `Resources` 文件夹中存在 `Dockerfile`：

```powershell
# 进入项目目录
cd D:\WinDevEnvSetup

# 构建镜像（需要 10-30 分钟，取决于网络速度）
podman build -t arch-cpp-dev:latest -f "Resources\Dockerfile" "Resources"
```

**Dockerfile 说明**：
- 基于 Arch Linux 最新版本
- 预装工具：GCC, Clang, GDB, LLDB, CMake, Ninja, Git, Fish Shell
- 包含 vcpkg 包管理器和 codelldb 调试器
- 使用清华大学镜像源加速下载
- 创建非 root 用户 `devel`（密码：无，sudo 免密）

### 4.3 验证镜像是否导入成功

```powershell
# 列出所有镜像
podman images

# 应该能看到 arch-cpp-dev 镜像
```

### 4.4 测试运行容器

```powershell
# 启动容器并进入交互式 Shell
podman run -it --rm arch-cpp-dev:latest

# 在容器内测试编译器
gcc --version
clang++ --version

# 退出容器
exit
```

## 第五步：配置 VSCode 开发环境

### 5.1 导入 VSCode 配置文件

由于 VSCode 权限限制，配置文件需要手动导入：

1. 打开 **VSCode**
2. 点击左下角的 **齿轮图标 (⚙️)** → 选择 **"配置文件 (Profiles)"**
3. 点击 **"导入配置文件... (Import Profile...)"**
4. 在文件选择对话框中，导航到项目目录：
   ```
   D:\WinDevEnvSetup\Resources\Container.code-profile
   ```
5. 选择该文件并确认导入

### 5.2 配置文件包含的内容

导入的配置文件预配置了以下内容：
- **扩展插件**：C/C++, CMake Tools, Dev Containers, CodeLLDB 等
- **容器连接设置**：自动识别 Podman 容器
- **编辑器设置**：代码格式化、智能提示等
- **调试配置**：GDB/LLDB 调试器配置

### 5.3 连接到容器

1. 在 VSCode 中按 `F1` 打开命令面板
2. 输入并选择：`Dev Containers: Attach to Running Container...`
3. 选择 `arch-cpp-dev` 容器
4. VSCode 会在新窗口中打开容器内的开发环境

### 5.4 （可选）手动安装必要扩展

如果配置文件导入失败，可手动安装以下扩展：

```
必需扩展：
- ms-vscode.cpptools (C/C++)
- ms-vscode.cmake-tools (CMake Tools)
- ms-vscode-remote.remote-containers (Dev Containers)
- vadimcn.vscode-lldb (CodeLLDB)

推荐扩展：
- ms-vscode.cpptools-extension-pack (C/C++ Extension Pack)
- twxs.cmake (CMake Language Support)
```

## 🔧 故障排查

### 问题 1：WSL 功能启用失败

**症状**：执行 `dism.exe` 命令时提示"找不到该功能"

**解决方案**：
1. 确认 Windows 版本是否满足要求（Win10 1903+ 或 Win11）
2. 检查 Windows 更新，安装所有待更新补丁
3. 尝试通过"控制面板"手动启用：
   - 打开"控制面板" → "程序" → "启用或关闭 Windows 功能"
   - 勾选"适用于 Linux 的 Windows 子系统"和"虚拟机平台"
   - 点击"确定"并重启

### 问题 2：Scoop 安装失败或速度慢

**症状**：安装脚本卡住或报网络错误

**解决方案**：
1. 检查网络连接，确保能访问 GitHub
2. 尝试使用代理：
   ```powershell
   $env:HTTP_PROXY = "http://127.0.0.1:7890"
   $env:HTTPS_PROXY = "http://127.0.0.1:7890"
   ```
3. 手动下载 Scoop 安装脚本：
   ```powershell
   Invoke-WebRequest -Uri "https://raw.githubusercontent.com/ScoopInstaller/Install/master/install.ps1" -OutFile "install_scoop.ps1"
   .\install_scoop.ps1 -RunAsAdmin
   ```

### 问题 3：Podman 虚拟机启动失败

**症状**：执行 `podman machine start` 时报错

**解决方案**：
1. 检查 WSL2 是否正常工作：
   ```powershell
   wsl --list --verbose
   ```
2. 重置 Podman 虚拟机：
   ```powershell
   podman machine stop
   podman machine rm
   podman machine init --rootful
   podman machine start
   ```
3. 检查 Hyper-V 是否与其他虚拟化软件冲突（如 VirtualBox、VMware）

### 问题 4：容器镜像构建失败

**症状**：执行 `podman build` 时报网络错误或包下载失败

**解决方案**：
1. 确保网络连接稳定
2. 检查 Dockerfile 中的镜像源是否可访问
3. 尝试更换镜像源（编辑 `Resources/Dockerfile`）：
   ```dockerfile
   # 将清华源替换为其他镜像源
   RUN echo "Server = https://mirrors.ustc.edu.cn/archlinux/\$repo/os/\$arch" > /etc/pacman.d/mirrorlist
   ```
4. 分步构建，逐行执行 Dockerfile 中的命令以定位问题

### 问题 5：VSCode 无法连接到容器

**症状**：VSCode 提示"无法连接到容器"

**解决方案**：
1. 确认容器正在运行：
   ```powershell
   podman ps -a
   ```
2. 手动启动容器：
   ```powershell
   podman run -d --name dev-container arch-cpp-dev:latest sleep infinity
   ```
3. 检查 VSCode 扩展是否正确安装：
   - 确保已安装 "Dev Containers" 扩展
   - 重启 VSCode
4. 检查 Podman 路径是否在系统 PATH 中：
   ```powershell
   $env:Path
   ```

### 问题 6：磁盘空间不足

**症状**：安装过程中提示磁盘空间不足

**解决方案**：
1. 清理 C 盘临时文件：
   ```powershell
   # 清理 Windows 临时文件
   Remove-Item -Path "$env:TEMP\*" -Recurse -Force -ErrorAction SilentlyContinue

   # 清理 Scoop 缓存
   scoop cache rm *
   ```
2. 检查并删除旧的 WSL 发行版：
   ```powershell
   wsl --list
   wsl --unregister <发行版名称>
   ```
3. 清理 Docker 残留文件（如果之前安装过）：
   ```
   %LocalAppData%\Docker\wsl
   %LocalAppData%\Packages\CanonicalGroupLimited.*
   ```

## 📝 环境变量检查清单

安装完成后，确认以下环境变量已正确设置：

```powershell
# 检查用户级环境变量
[Environment]::GetEnvironmentVariable('SCOOP', 'User')
# 应输出: D:\Scoop

# 检查系统级环境变量
[Environment]::GetEnvironmentVariable('SCOOP_GLOBAL', 'Machine')
# 应输出: D:\ScoopGlobal

# 检查 PATH 是否包含 Scoop
$env:Path -split ';' | Select-String "Scoop"
# 应包含: D:\Scoop\shims

# 验证命令可用性
Get-Command scoop, podman, code -ErrorAction SilentlyContinue
# 应显示三个命令的路径
```

## ✅ 安装完成验证

执行以下命令验证环境是否配置成功：

```powershell
# 1. 检查 WSL 版本
wsl --version

# 2. 检查 Scoop 版本
scoop --version

# 3. 检查 Podman 版本
podman --version

# 4. 检查 Podman 虚拟机状态
podman machine list

# 5. 检查容器镜像
podman images

# 6. 运行测试容器
podman run -it --rm arch-cpp-dev:latest gcc --version
```

如果以上命令均正常执行，说明环境配置成功！
